Section 2
Middleware & Access Control
File: src/middleware.ts — Runs before every matched request, handles auth redirects and role-based routing.
Important: API routes (/api/*) bypass the middleware entirely. Each API handler performs its own authentication check internally via auth() or getServerSession().
Middleware Route Matchers
Protected Path PatternRequired RoleRedirect If UnauthenticatedNotes
/admin · /admin/*ADMIN / SUPPORT_AGENT / SALES_AGENT / CONTENT_MANAGER/loginWrong role → redirect to own dashboard
/dashboard · /dashboard/*Any authenticated session/login—
/account · /account/*Any authenticated session/login—
/vendor/* (excl. /vendor/register)VENDOR role/login/vendor/register is public
/login · /registerUnauthenticated onlyRole-based redirect on re-visitLogged-in users are bounced away

LayerComponentResponsibility
