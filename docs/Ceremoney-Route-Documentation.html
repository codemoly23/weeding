Legend & Reference

Color codes, badge meanings, and notation used throughout this document

🔐 Access Level Badges
PUBLIC No login required — anyone can access
AUTH Any logged-in user
PLANNER Logged-in customer with planner access
VENDOR Registered vendor role only
ADMIN Admin or super_admin role
GUEST Event guest — no account, token-based
⚡ HTTP Method Badges
GET Retrieve data — safe, cacheable
POST Create new resource
PUT Replace entire resource
PATCH Partial update of resource
DELETE Remove a resource
📝 Path Notation
[id] — Dynamic segment (any value)
[slug] — URL-friendly string identifier
[token] — Secure share token
[...path] — Catch-all segments
(group) — Next.js route group (not in URL)
📁 Route Groups (Next.js)
(marketing) Public-facing site
(auth) Login / Register pages
dashboard/ Customer portal
planner/ Event planning suite
admin/ Admin control panel
vendor/ Vendor business portal
🗝️ Special Tokens
RSVP slug — Human-readable couple identifier
Share token — Cryptographic project share URL
Brief token — Vendor event brief access token
Project ID — Cuid-format database identifier
Webhook secret — Stripe/PayPal event verification
⚠️ Auth Notes
All API routes return 401 for unauthenticated requests where auth is required
Role mismatch returns 403 Forbidden
Anonymous planner data stored in localStorage, synced on login
Vendor routes require admin approval first

Key Design Principle: Anonymous-First
All planner page routes work without authentication using localStorage. The API layer (/api/planner/) is auth-protected, but the UI degrades gracefully to local storage — so users can start planning with zero friction. The /planner/sync route handles the one-time migration when a user registers.
