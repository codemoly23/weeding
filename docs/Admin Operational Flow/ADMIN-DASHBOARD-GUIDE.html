What This Controls

This page controls exactly what options customers see when they search on the homepage. The three columns — Place, Service, Location — fill the three homepage search dropdowns.

How to Add / Edit / Delete an Option

  • 1Type the name in the "Add" input field at the top of the column
  • 2Optionally add an icon name (Lucide icon library)
  • 3Click the "Add" button — item appears in the list below

  • 5To delete: hover over the item — an ✕ button appears on the right — click it

Role Permissions Page  /admin/users/permissions

Click "Role Permissions" button (top-right of Users page) to access granular permission control.

  • 1Select the role tab: Content Manager / Sales Agent / Support Agent
  • 2Each category (e.g., "Content Management") shows a checkbox list of specific permissions
  • 3Check/uncheck individual permissions. "Select All / Deselect All" button per category for bulk toggle.
  • 4Orange dot badge = unsaved changes. Click "Save Changes" to apply.
  • 5Changes apply to ALL users with that role immediately.
⚠️
Admin and Customer roles are locked — they cannot be edited from this page.

Payment Settings  /admin/settings/payments

🔒
Always use Test Mode when setting up. Switch to Live Mode only after successful test transaction. Never share API keys.

Stripe Setup Steps

  • 1Toggle "Enable Stripe" ON
  • 2Set Mode to "Test Mode"
  • 3Paste Publishable Key (pk_test_...)
  • 4Paste Secret Key (sk_test_...)
  • 5Paste Webhook Secret (whsec_...)
  • 6Copy the Webhook URL shown → paste in Stripe Dashboard
  • 7Click "Test Connection" → should show green Connected
  • 8When ready for live: switch Mode to "Live Mode" → replace keys

PayPal Setup Steps

  • 1Toggle "Enable PayPal" ON
  • 2Set Mode to "Sandbox Mode"
  • 3Paste Client ID and Client Secret
  • 4Paste Webhook ID
  • 5Copy Webhook URL → paste in PayPal Developer Dashboard
  • 6Click "Test Connection"

📋 Daily Admin Checklist

Run through this every morning. Takes 10–15 minutes.

Morning (Start of Day)

  • Check dashboard stat cards for anomalies
  • Process overnight pending orders
  • Resolve any FAILED payment orders
  • Review urgent/high priority tickets
  • Check pending vendor applications

During Day

  • Monitor live chat queue
  • Respond to open tickets (<4 hrs)
  • Process incoming orders
  • Review any new vendor signups
  • Check email delivery health

End of Day

  • Ensure no tickets older than 8 hrs unresolved
  • Review today's orders summary
  • Check Vercel deployment logs for errors
  • Review any scheduled campaigns for tomorrow
  • Export daily report if needed
