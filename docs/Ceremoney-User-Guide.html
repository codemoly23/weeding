Section 01

Platform Overview

What Ceremoney is, who it's built for, and the problem it solves

✦
What is Ceremoney?
Ceremoney is a full-stack SaaS platform that enables couples, professional event planners, and vendors to manage every dimension of an event — from the first guest invite to the last vendor payment. It operates across three distinct user portals: a Public Marketing Site, a Customer/Planner Dashboard, and a Vendor Portal — all managed by a powerful Admin Control Panel.
🎯
Primary Target Markets
Sweden (Primary) — Swedish language, Swish payments, SEK currency, 25% VAT compliance

Global English — USD pricing, Stripe & PayPal, international vendor directory

Arabic (AR) — Full RTL layout support, Arabic typography (Cairo font)
📅
Supported Event Types
🥂 Wedding — Full suite of planning tools

✝️ Baptism — Ceremony coordination

🎉 Party — Guest & venue management

🏢 Corporate — Professional event management

Each event type shares the same core modules but can be configured independently.

User Roles & Access Levels

Six distinct roles with carefully scoped capabilities and permissions

Role Access Level Key Capabilities Login Required
super_admin Full Platform All settings, financials, user management, plugin management, vendor approvals, data reset ✅ Yes
admin Operational User support, content moderation, order management, vendor oversight, lead pipeline ✅ Yes
vendor Own Business Edit business profile, view inquiries, real-time messaging, availability calendar, reviews ✅ Yes
customer Own Projects Create & manage events, guests, website, budget, checklist, vendor coordination, billing ✅ Yes
collaborator Shared Project View/edit guest list with restricted permissions — granted by host (Elite plan only) ✅ Yes
guest RSVP Only Submit RSVP form, view assigned seat via QR code, sign guestbook, upload photos ❌ No account needed
💡
Anonymous Mode — Start Without Registering
A key differentiator: users can begin planning immediately in the browser without creating an account. All data is saved locally using the browser's storage. When they register later, all data syncs automatically to the cloud — zero data loss.

Section 14

Technology & Architecture

The technical foundation that powers Ceremoney's reliability and performance

Next.js 15 (App Router)
Server Components for SEO & performance. React for interactive UI. Deployed on Vercel with global CDN.
TypeScript 5.9
Full type safety across the entire codebase — frontend, backend, API routes, and database queries.
PostgreSQL + Prisma
95+ database models on AWS RDS. Prisma ORM for type-safe queries and schema migrations.
Tailwind CSS 4
Utility-first styling with a 4px spacing system. Mobile-first responsive design. Full RTL support.
Socket.io
Real-time bidirectional communication for live vendor-couple messaging and collaborative planning.
Konva.js
Canvas-based interactive seating chart editor with drag-and-drop, zoom, and conflict detection.
AWS S3 + CloudFront
File uploads via presigned URLs. Global CDN delivery for photos and documents.
Stripe + PayPal + Swish
Multi-gateway payments. Stripe for subscriptions & invoicing. Swish for Sweden. Klarna for BNPL.
next-intl (i18n)
Swedish, English, and Arabic localization. Full RTL layout for Arabic with Cairo typography.
🏗️
Deployment Architecture
Frontend: Vercel (Next.js, global edge network)  ·  API: AWS ECS (containerized)  ·  Database: AWS RDS PostgreSQL  ·  Storage: AWS S3 + CloudFront CDN  ·  Real-time: Socket.io server (separate process)
